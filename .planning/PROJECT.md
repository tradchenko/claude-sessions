# claude-sessions: Аудит и стабилизация

## What This Is

CLI-утилита для управления сессиями AI-агентов (Claude Code, Codex, Qwen, Gemini, Companion). Позволяет просматривать, возобновлять, восстанавливать сессии и извлекать из них «память» — структурированные воспоминания, которые подгружаются в новые сессии. Установлена у пользователей через npm, работает с файлами в домашних директориях агентов.

## Core Value

Каждая команда из README должна работать надёжно и предсказуемо для всех поддерживаемых агентов, включая edge cases и обновление с предыдущих версий.

## Context

- Проект brownfield, ~42 TypeScript файла, ~2100 строк маппинга кодовой базы
- Пользователи уже установили утилиту через npm (`npm i -g claude-sessions`)
- Хуки в `~/.claude/settings.json` имели ломающие изменения формата в прошлом
- 5 агент-адаптеров с дублирующейся логикой
- Монолитный `i18n.ts` (2132 строк)
- Тестовое покрытие минимальное — нет `*.test.ts` файлов
- Память использует L0 (quick metadata) и L1 (LLM semantic) extraction
- Сессии хранятся в разных форматах: Claude (sessions.json), Codex (JSONL в ~/.codex/), Qwen (JSONL в ~/.qwen/), Gemini (git repos), Companion (JSONL)

## Constraints

- **Backward compat**: npm update не должен ломать существующую установку, миграция автоматическая
- **Zero deps**: Runtime-зависимости не добавляем
- **Cross-platform**: macOS, Linux, Windows WSL
- **Node.js ≥18**: ES2022 target, ESM modules
