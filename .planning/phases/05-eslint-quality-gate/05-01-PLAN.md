# Plan 05-01: ESLint strict config, Prettier, quality gate scripts

## Goal

Мигрировать ESLint на unified API с strict правилами, создать .prettierrc, добавить quality gate скрипты (lint, lint:fix, typecheck, check) и исправить все нарушения — обеспечив 0 errors при `npm run check`.

## must_haves

- [ ] eslint.config.mjs с typescript-eslint/strict для src/ и базовыми правилами для tests/*.mjs
- [ ] .prettierrc с tabWidth=3, singleQuote=true, printWidth=180
- [ ] npm run lint — 0 errors на src/ и tests/
- [ ] npm run check (lint + typecheck) — 0 errors
- [ ] npm run build — не сломан после всех изменений
- [ ] eslint-config-prettier последним в конфиге (нет конфликтов с Prettier)

<task id="05-01-03" title="Мигрировать ESLint конфиг на unified API + strict" autonomous="true">

### Description

Заменить `eslint.config.js` на `eslint.config.mjs` с unified API:

1. Удалить старый `eslint.config.js`
2. Создать `eslint.config.mjs` со структурой:
   - Глобальные ignores: `dist/`, `build/`, `node_modules/`, `.planning/`, `.claude/`
   - Блок src/*.ts: `tseslint.configs.strict`, `eslint-plugin-import` правила, `no-console: off`
   - Блок tests/*.mjs: базовые правила без TypeScript parser (JS-файлы)
   - `eslint-config-prettier` последним
3. Настроить правила import: порядок группировки (builtin → external → internal)
4. `no-console: off` — CLI-инструмент, console допустим

### Acceptance Criteria

- [ ] `eslint.config.mjs` существует
- [ ] `eslint.config.js` удалён
- [ ] typescript-eslint/strict применяется к src/**/*.ts
- [ ] tests/**/*.mjs линтятся базовыми правилами
- [ ] eslint-config-prettier последним в цепочке
- [ ] Ignores включают dist/, build/, node_modules/, .planning/, .claude/
