# Plan 02-07: Unit тесты на адаптеры с DI

## Контекст

Текущие тесты адаптеров в test/agents.test.mjs проверяют только registry (detect структуры), но не поведение loadSessions(), getResumeCommand(), isSessionAlive(). После DI через конструктор (CODE-03) можно мокать файловую систему и тестировать каждый адаптер изолированно.

## Подход

Fixture-based матрица [5 агентов] × [valid/empty/missing/corrupted/resume] с мокнутым FsDeps. Продолжать паттерны из Phase 1.
