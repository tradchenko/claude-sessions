# Plan 02-05: Restore — мультиагентность и идемпотентность

## Контекст

restore.ts полностью захардкожен на Claude: findClaudeCli(), CLAUDE_DIR, execFileSync(claudePath). Нет поддержки других агентов. Файл .restore-context.md перезаписывается при каждом вызове (нет идемпотентности).

## Подход

Полная переработка restore.ts:
1. Определить агент по session.agent → получить адаптер из registry
2. Путь контекстного файла — в директории проекта (не в ~/.claude/)
3. Идемпотентность: проверить существование файла перед созданием
