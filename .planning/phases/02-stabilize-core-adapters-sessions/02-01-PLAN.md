# Plan 02-01: Фундамент — DI, BaseAgentAdapter, AdapterError

## Контекст

Текущие адаптеры — object literals (singletons). Файловая система не инжектируется, мокать невозможно. Нет базового класса, нет структурированных ошибок адаптеров. Это фундамент для всех остальных планов Phase 2.

## Подход

1. Определить интерфейс `FsDeps` (readFile, readdir, stat) и тип `AdapterErrorCode`
2. Создать класс `AdapterError extends Error` с полями agentName, code, suggestion
3. Создать абстрактный класс `BaseAgentAdapter` с DI конструктором и общими методами
4. Переписать 4 адаптера (Claude, Codex, Qwen, Gemini) как классы наследующие Base
5. Companion — отдельная реализация AgentAdapter через композицию (не наследует Base)
6. Обновить registry для работы с новыми классами (factory-функции)
