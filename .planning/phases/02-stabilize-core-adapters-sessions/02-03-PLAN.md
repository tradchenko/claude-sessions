# Plan 02-03: Resume для всех 5 агентов

## Контекст

Claude resume работает. Codex явно возвращает null. Qwen и Gemini — неизвестно. Companion делегирует, но может сломаться. Нет обработки "сессия не найдена" и "агент не установлен" с понятными ошибками.

## Стратегия для агентов без --resume

- Если агент не поддерживает --resume нативно → `getResumeCommand()` возвращает команду запуска агента в директории проекта (cd + запуск). Если и это невозможно — бросает `AdapterError.resumeNotSupported()`.
- Для Codex: открыть codex в projectPath (если известен), иначе AdapterError
- Для Qwen/Gemini: проверить поддерживает ли CLI --resume, если нет — аналогично Codex

<task id="02-03-04">
<title>Обработка ошибок resume в команде и picker</title>
<what>
В команде resume (`src/commands/resume.ts` или аналог):
- При вызове adapter.getResumeCommand(sessionId):
  - Если результат null → бросить AdapterError.resumeNotSupported(agentName)
  - Если AdapterError.SESSION_NOT_FOUND → показать структурированную ошибку (formatUserError), exit code 2
  - Если AdapterError.AGENT_NOT_INSTALLED → показать ошибку, exit code 3
  - Если AdapterError.RESUME_NOT_SUPPORTED → показать ошибку с suggestion "restore", exit code 1

В picker (`src/commands/picker.ts`):
- При ошибке resume внутри picker: показать сообщение в status bar, НЕ выходить из picker
- Пользователь может выбрать другую сессию
</what>
