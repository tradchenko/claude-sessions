# Plan 02-06: TUI picker — SIGWINCH и стабильность навигации

## Контекст

Picker реализован на raw ANSI (не ink). Alt screen, mouse, навигация работают. Нет обработки SIGWINCH — при ресайзе терминала артефакты. При ошибке resume поведение неясно.

---

<task id="02-06-01">
<title>Добавить обработку SIGWINCH</title>
<what>
В `src/commands/picker.ts` (класс SessionPicker):
- Добавить обработчик `process.on('SIGWINCH', handleResize)`:
  - Пересчитать rows/columns из process.stdout
  - Пересчитать видимую область (pageSize, scrollOffset)
  - Вызвать полную перерисовку (clearScreen + render)
- В cleanup() обязательно снять SIGWINCH listener: `process.removeListener('SIGWINCH', handleResize)`
- Предотвратить memory leak: проверить что cleanup вызывается при любом выходе (Enter, Esc, Ctrl-C)
- Debounce: если resize приходит чаще 100ms → перерисовать только последний
</what>

<task id="02-06-02">
<title>Стабилизировать обработку ошибок внутри picker</title>
<what>
В `src/commands/picker.ts`:
- При ошибке resume (getResumeCommand() бросает AdapterError или возвращает null):
  - НЕ выходить из picker
  - Показать сообщение об ошибке в status bar (нижняя строка picker'а)
  - Формат: `⚠ {adapter.formatError(error)}` — краткая версия без трёхчастного формата
  - Через 3 секунды или при следующем действии пользователя — убрать сообщение
  - Пользователь может выбрать другую сессию
- При критической ошибке (не AdapterError) — выйти из picker, показать полную ошибку через handleFatalError()
</what>

<task id="02-06-03">
<title>Проверить навигацию и граничные случаи</title>
<what>
В `src/commands/picker.ts`:
- Проверить что ↑↓ навигация корректна при: 0 сессий, 1 сессия, 100+ сессий
- Tab фильтр: если после фильтрации 0 сессий → показать сообщение, не crash
- Enter на пустом списке → ничего не делать (не crash)
- Проверить что scroll offset корректен после resize (не выходит за границы)
- Проверить что mouse click корректен после resize (пересчёт позиций)
- Граничный случай: терминал width < 40 → минимальный render без обрезки
