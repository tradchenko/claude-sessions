# Plan 04-03: Unit тесты миграции

## Objective
Написать unit тесты для всей системы миграции: backup, runMigrations(), идемпотентность, обработка ошибок. Покрыть все MIG-* сценарии тестами через DI (temp directories).

## must_haves
- Тесты `backupFile()`: создание backup, cleanup старых копий
- Тесты `runMigrations()`: v0→v1 миграция, идемпотентность (повторный вызов = no-op), обработка ошибок
- Тесты `migrateHooks()`: flat format → wrapped, уже мигрированный формат не ломается
- Тесты `readMeta()` / `writeMeta()`: чтение, запись, отсутствующий файл
- Все тесты используют temp directories и cleanup
- `npm test` зелёный

Создать секцию тестов backup в `tests/migration.test.mjs`:

4. **readMeta() / writeMeta():**
   - Чтение несуществующего файла → null
   - Запись и чтение → данные совпадают
   - Невалидный JSON → null
