# Plan 04-02: Lazy migration в CLI и error handling

## Objective
Добавить вызов `runMigrations()` в `cli.ts` перед dispatch команд. Это обеспечивает миграцию при `--ignore-scripts` (когда postinstall не запускался). Гарантировать что ошибки миграции не блокируют работу CLI.

## must_haves
- `cli.ts` вызывает `runMigrations({ silent: false })` перед dispatch любой команды
- При отсутствии `meta.json` или устаревшей версии — миграция выполняется автоматически
- Если `meta.json` актуален — no-op без задержки
- Ошибка миграции не блокирует выполнение команды — CLI продолжает работу
- Пользователь видит краткое сообщение о миграции (одна строка)

## Tasks

<task id="04-02-01">
<title>Lazy migration check в cli.ts</title>
<action>
Модифицировать `src/cli.ts`:
1. Импортировать `runMigrations` из `src/migration`
2. Перед switch/case dispatch добавить:
   ```ts
   // Lazy миграция — для случая --ignore-scripts
   await runMigrations({
      claudeDir: CLAUDE_DIR,
      dataDir: DATA_DIR,
      silent: false,
   });
   ```
3. Вызов должен быть ДО парсинга команды, чтобы все команды работали с актуальным форматом данных
4. `runMigrations()` уже обрабатывает ошибки внутри (MIG-06) — дополнительный try/catch не нужен
</action>
