# Plan 04-01: Система миграции: runner, backup, meta.json

## Objective
Создать единую функцию `runMigrations()` с версионированием через `meta.json`, backup конфигов перед миграцией, atomic writes и идемпотентностью. Обернуть существующие `migrateHooks()` и `migrateSessionIndex()` в новый runner. Postinstall использует `runMigrations()` вместо прямых вызовов.

## must_haves
- `meta.json` создаётся в `~/.claude-sessions/` с `schema_version` после миграции
- `runMigrations()` идемпотентна — повторный вызов при актуальной версии = no-op
- Backup файлов (`settings.json`, `memory-index.json`) перед изменением в `~/.claude-sessions/backups/`
- Хранение максимум 3 backup-копий, старые удаляются
- Atomic write для `settings.json` и `meta.json` (temp + rename)
- При ошибке миграции — логирование, graceful return (exit 0), не throws
- `install.ts` postinstall вызывает `runMigrations()` вместо прямых вызовов миграции

<task id="04-01-03">
<title>Функция runMigrations()</title>
<action>
Создать `src/migration/runner.ts`:
1. Функция `readMeta(dataDir: string): MetaJson | null` — читает `meta.json`, при ошибке возвращает null (= v0)
2. Функция `writeMeta(dataDir: string, meta: MetaJson): void` — atomic write `meta.json` (temp + rename)
3. Массив миграций `MIGRATIONS: Migration[]`:
   - `{ fromVersion: 0, toVersion: 1, migrate }` — вызывает `migrateHooks()` и `migrateSessionIndex()` из существующего кода
4. Функция `runMigrations(ctx: MigrationContext): Promise<MigrationResult>`:
   - Читает `meta.json` → определяет текущую версию (null = 0)
   - Если `schema_version >= CURRENT_SCHEMA_VERSION` → early return (идемпотентность, MIG-03)
   - Вызывает `backupFile()` для `settings.json` и `memory-index.json` перед миграцией (MIG-07)
   - Фильтрует `MIGRATIONS` где `fromVersion >= currentVersion`, выполняет последовательно
   - Записывает `meta.json` с новой версией (MIG-05 — atomic)
   - Обёрнуто в try/catch: при ошибке логирует и возвращает `{ ok: false, error }` (MIG-06)
   - Если `!silent` — выводит краткое сообщение о миграции
5. Экспорт: `runMigrations`, `readMeta`, `CURRENT_SCHEMA_VERSION`
</action>
