# Plan 01: TypeScript strict mode, ESLint и shared utilities

## Objective
Включить `noUncheckedIndexedAccess` в tsconfig, настроить ESLint, вынести дублирующуюся логику (JSONL-парсинг, безопасное чтение файлов, нормализация путей) в `src/utils/` с Result type для error handling. Исправить все TS-ошибки, возникшие от strict-режима.

## must_haves
- `npx tsc --noEmit` проходит без ошибок с `noUncheckedIndexedAccess: true`
- ESLint настроен с `@typescript-eslint/recommended`, `npm run lint` работает
- `src/utils/` содержит `jsonl.ts`, `fs.ts`, `paths.ts`, `result.ts` с Result type
- Все адаптеры используют shared utilities вместо дублированного кода
- Ни одна TS-ошибка не подавлена через `@ts-ignore`

<task id="01-01-01" title="Включить noUncheckedIndexedAccess и исправить TS-ошибки">

**Do:** В `tsconfig.json` добавить `"noUncheckedIndexedAccess": true`. Запустить `npx tsc --noEmit`, собрать список ошибок. Исправить каждую ошибку через proper narrowing (optional chaining, type guards, nullish coalescing) — НЕ через `@ts-ignore` или `as` casts. Все комментарии в коде — на русском.

**Files:** tsconfig.json, все `.ts` файлы в `src/` где возникнут ошибки

<task id="01-01-02" title="Создать Result type и модуль src/utils/">

**Do:** Создать директорию `src/utils/` с файлами:
- `result.ts` — тип `Result<T> = { ok: true; data: T } | { ok: false; error: string }` и helper-функции `ok(data)`, `err(error)`
- `jsonl.ts` — функция `parseJsonlLines(content: string): Result<unknown[]>` с обработкой BOM, пустых строк, невалидного JSON. Функция `parseJsonlFile(filePath: string): Result<unknown[]>` для чтения и парсинга
- `fs.ts` — функция `safeReadFile(path: string): Result<string>`, `safeReadJson<T>(path: string): Result<T>`, `safeReaddir(path: string): Result<string[]>`
- `paths.ts` — функция `normalizePath(p: string): string` для кросс-платформенной нормализации
- `index.ts` — re-exports из всех модулей

Prettier: tabWidth=3, singleQuote=true, printWidth=180. Комментарии на русском. Функциональный/иммутабельный стиль.

**Files:** src/utils/result.ts, src/utils/jsonl.ts, src/utils/fs.ts, src/utils/paths.ts, src/utils/index.ts

<task id="01-01-03" title="Рефакторинг адаптеров на shared utilities">

**Do:** В каждом из 5 адаптеров (`claude.ts`, `codex.ts`, `gemini.ts`, `qwen.ts`, `companion.ts`) и в `sessions/loader.ts` заменить дублированные паттерны:
- `JSON.parse` в try/catch циклах → `parseJsonlFile()` или `parseJsonlLines()`
- `readFileSync` + JSON.parse → `safeReadJson()`
- `existsSync` + `readdirSync` → `safeReaddir()`
- Inline path normalization → `normalizePath()`

Убедиться что внешнее поведение НЕ меняется — только внутренняя реализация. Комментарии на русском.

**Files:** src/agents/claude.ts, src/agents/codex.ts, src/agents/gemini.ts, src/agents/qwen.ts, src/agents/companion.ts, src/sessions/loader.ts
