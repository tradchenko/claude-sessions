# Plan 02: Fixture-файлы и тесты JSONL edge cases

## Objective
Создать fixture-файлы для всех JSONL edge cases и написать тесты, фиксирующие текущее поведение парсера: невалидные строки, обрезанные файлы, BOM, null-поля, пустые/отсутствующие директории, большие файлы. Каждый DATA-requirement покрыт минимум одним тестом.

## must_haves
- Fixture-файлы существуют в `tests/fixtures/shared/` и `tests/fixtures/{agent}/`
- Каждый DATA-01..07 покрыт тестом с fixture
- 10MB+ файл генерируется на лету test helper-ом (не коммитится)
- `npm test` запускает все новые тесты и проходит
- Тесты документируют ТЕКУЩЕЕ поведение (snapshot before refactor)

<task id="01-02-01" title="Создать fixture-файлы для JSONL edge cases">

**Do:** Создать директории `tests/fixtures/shared/`, `tests/fixtures/claude/`, `tests/fixtures/codex/`, `tests/fixtures/qwen/`, `tests/fixtures/companion/`. Создать файлы:

**shared/**
- `truncated.jsonl` — 3 валидных строки + 1 обрезанная (`{"sessionId": "abc", "timesta`)
- `bom.jsonl` — файл начинается с UTF-8 BOM (`\uFEFF`) + 2 валидных JSONL строки
- `null-fields.jsonl` — строки с `"sessionId": null`, `"timestamp": null`, `"project": null`
- `empty.jsonl` — пустой файл (0 байт)
- `invalid-lines.jsonl` — микс валидных и невалидных строк (plain text, XML, пустые)

**claude/**
- `valid-history.jsonl` — 5 строк формата `{"sessionId":"uuid","project":"/path","display":"...","timestamp":1234567890000}`
- `valid-session.jsonl` — 10 строк анонимизированной Claude-сессии

**codex/**
- `valid-history.jsonl` — 5 строк формата `{"session_id":"...","ts":1234567890,"text":"..."}`

**qwen/**
- `valid-session.jsonl` — 5 строк формата `{"sessionId":"...","timestamp":...,"cwd":"..."}`

**companion/**
- `valid-recording.jsonl` — заголовок `_header` + 3 строки записи

Все fixture-файлы — синтетические, с анонимизированными данными. Комментарий в начале каждого файла не нужен (JSONL не поддерживает комментарии).

**Files:** tests/fixtures/shared/truncated.jsonl, tests/fixtures/shared/bom.jsonl, tests/fixtures/shared/null-fields.jsonl, tests/fixtures/shared/empty.jsonl, tests/fixtures/shared/invalid-lines.jsonl, tests/fixtures/claude/valid-history.jsonl, tests/fixtures/claude/valid-session.jsonl, tests/fixtures/codex/valid-history.jsonl, tests/fixtures/qwen/valid-session.jsonl, tests/fixtures/companion/valid-recording.jsonl

**Do:** Создать `tests/helpers/generate-large-jsonl.mjs` — экспортирует функцию `generateLargeJsonl(targetSizeMB = 10)` которая возвращает путь к временному файлу с JSONL нужного размера. <task id="01-02-03" title="Написать тесты JSONL-парсера на fixture-файлах">

**Do:** Создать `tests/jsonl-parser.test.mjs` с тестами через `node:test` + `node:assert`:

- **DATA-01:** Парсинг `invalid-lines.jsonl` — невалидные строки пропускаются, валидные возвращаются
- **DATA-04:** Парсинг `truncated.jsonl` — обрезанная строка пропускается, остальные парсятся
- **DATA-05:** Парсинг `bom.jsonl` — BOM не мешает парсингу первой строки
- **DATA-06:** Парсинг `null-fields.jsonl` — строки с null полями не крэшат парсер

Тесты самодостаточны: парсят JSONL напрямую через `fs.readFileSync` + `JSON.parse` построчно. НЕ импортируют из `dist/` или shared utils — тестируют логику парсинга inline, без зависимости от Plan 01.

**Files:** tests/jsonl-parser.test.mjs

<verify>
<automated>
```bash

```
</automated>
</verify>
