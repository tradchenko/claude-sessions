# Plan 03: Session identity тесты и fixture-based матрица

## Objective
Задокументировать session identity каждого из 5 агентов через проходящие тесты (тесты-как-спецификация). Создать fixture-based тестовую матрицу [agent] x [command] x [edge case], покрывающую основные сценарии. Conditional тесты на реальных данных машины скипаются если агент не установлен.

## must_haves
- Для каждого из 5 агентов есть тест, документирующий: формат JSONL, извлечение session ID, путь хранения
- Fixture-based матрица покрывает: [5 agents] x [loadSessions] x [valid, empty, missing, truncated]
- Conditional тесты на реальных данных скипаются в CI
- `npm test` запускает все тесты и проходит
- `claude-sessions list` не падает при повреждённых данных (интеграционная проверка)

<tasks>

<task id="01-03-01" title="Написать session identity тесты для каждого агента">

**Do:** Создать `tests/session-identity.test.mjs` с тестами через `node:test`:

Для каждого агента (claude, codex, gemini, qwen, companion) — отдельный `describe` блок:

1. **Формат хранения** — тест парсит fixture-файл агента, проверяет что ожидаемые поля присутствуют (sessionId/session_id, timestamp/ts, project/cwd)
2. **Извлечение session ID** — тест вызывает `loadSessions()` на mock-директории с fixture, проверяет что ID извлекается корректно
3. **Путь хранения** — тест проверяет что адаптер ищет файлы в ожидаемых путях (через mock homedir)
4. **Conditional: реальные данные** — `test` с `{ skip: !agentDetected }`: если агент установлен на машине, вызвать `loadSessions()` на реальных данных и проверить что результат — массив (может быть пустой)

Комментарии на русском. Тесты импортируют из `dist/`.

**Files:** tests/session-identity.test.mjs

<task id="01-03-02" title="Создать fixture-based тестовую матрицу">

**Do:** Создать `tests/fixture-matrix.test.mjs` — матричные тесты через `node:test`:

Матрица: [agent] x [scenario]:
- **valid** — fixture валидной сессии → `loadSessions()` возвращает >=1 сессию
- **empty** — пустой fixture → возвращает пустой массив
- **missing-dir** — директория не существует → возвращает пустой массив, не падает
- **truncated** — truncated.jsonl в директории агента → парсит что может, не падает

Для каждого агента создать временную директорию со структурой, имитирующей реальную файловую систему агента, подложить fixture-файлы, вызвать `loadSessions()`.

Если адаптер требует конкретную структуру директорий — создать её в tmpdir. Комментарии на русском.

**Files:** tests/fixture-matrix.test.mjs
