# Plan 03-03: L1 аудит и memory loading при старте сессии

## Objective
Провести полный аудит L1 LLM extraction (prompt, парсинг, retry, edge cases), исправить найденные баги. Убедиться что hot memories корректно загружаются при старте сессии.

## Context
- Найденные проблемы в extract-l1.ts (из RESEARCH.md):
  1. Нет retry при ошибке spawnSync
  2. parseLLMResponse regex может поймать markdown [link] вместо JSON
  3. Gemini skip без записи l1_skipped флага
  4. Fuzzy match → skip вместо merge (фиксится в 03-02-04)
  5. Нет фильтра пустого content у кандидата
- session-start.ts — standalone скрипт, дублирует logiku catalog/hotness
- formatSessionStartOutput в catalog.ts — основная точка для memory loading

<task id="03-03-01" title="Аудит и фикс parseLLMResponse">
В `src/memory/extract-l1.ts`:
1. **Улучшить regex** для поиска JSON массива: вместо `\[[\s\S]*\]` использовать более точный паттерн — искать `[` после начала строки или после пробелов, игнорируя markdown ссылки `[text](url)`. Например: искать первый `[{` или `[\n{` паттерн.
2. **Фильтр пустого content**: после парсинга JSON отфильтровать кандидатов с пустым или whitespace-only `content`.
3. **Валидация схемы кандидата**: каждый кандидат должен иметь минимум `content` (непустой string) и `category` (string). Невалидные — отбрасывать с warning.
4. Добавить обработку случая когда LLM вернул пустой ответ → возвращать пустой массив `[]`.
</task>

<task id="03-03-02" title="Gemini l1_skipped флаг и retry logic">
В `src/memory/extract-l1.ts`:
1. **Gemini**: вместо `process.exit(0)` при `agentId === 'gemini'` — записать `l1_skipped: true` в `index.sessions[sessionId]` и завершить extraction для этой сессии (return, не exit). Это предотвратит повторные попытки extraction.
2. **Retry**: при `spawnSync` status !== 0 — добавить одну повторную попытку с паузой 2 секунды. Если и вторая попытка неудачна — записать ошибку и продолжить (не throw).
3. Добавить документирующий комментарий о HEAD_COUNT=15, TAIL_COUNT=35 как known limitation для длинных сессий.
</task>

<task id="03-03-03" title="Memory loading: formatSessionStartOutput тесты">
Убедиться что `formatSessionStartOutput` в `src/memory/catalog.ts`:
1. Корректно обрабатывает пустой индекс (нет memories) — возвращает пустую строку или placeholder
2. Корректно фильтрует hot memories по проекту
3. Форматирует вывод без ошибок для различных размеров catalog (0, 1, 30 memories)

В `src/hooks/session-start.ts`:
- Убедиться что hook вызывает catalog логику без ошибок
- Если индекс не существует — graceful exit (не crash)
</task>
