Покрыть unit тестами все модули подсистемы памяти (hotness, dedup, index, extract-l0, extract-l1, catalog). Проверить что README документация для memory команд соответствует реальному поведению.

<task id="03-05-02" title="Тесты hotness и dedup" depends_on="03-05-01">
Создать `tests/memory-hotness.test.mjs`:
- `calculateHotness` с датой 60 дней назад → recencyScore ≈ 0.368 (exp(-1))
- `calculateHotness` с датой сегодня → recencyScore ≈ 1.0
- `recalculateAll` — пересчёт для массива memories
- Проверить что DECAY_TAU_DAYS = 60
- Проверить веса frequency=0.4, recency=0.3, relevance=0.3

Создать `tests/memory-dedup.test.mjs`:
- `jaccardSimilarity` — одинаковые строки → 1.0, разные → 0.0, частичное совпадение → (0, 1)
- `findMatch` — exact match (similarity > 0.8) → action=skip
- `findMatch` — fuzzy match (similarity 0.5-0.8) → action=merge (не skip!)
- `findMatch` — no match (similarity < 0.5) → action=add
- `mergeContent` — объединяет уникальное из двух memories
- `resolveCandidate` — интеграционный тест resolve pipeline
- Проверить что FUZZY_THRESHOLD = 0.5
</task>

<task id="03-05-03" title="Тесты index, extract-l0, extract-l1, catalog" depends_on="03-05-01">
Создать `tests/memory-index.test.mjs`:
- `writeIndex` — после записи tmp файл отсутствует, index файл содержит корректный JSON
- `readIndex` — cleanup stale tmp файлов (создать tmp старше 5 минут, вызвать readIndex, проверить удаление)
- `acquireLock` / `releaseLock` — базовый lock/unlock цикл

Создать `tests/memory-extract-l0.test.mjs`:
- Fixture matrix: для каждого из 5 агентов — загрузить fixture, вызвать extraction, сравнить с expected.json
- Детерминированность: два вызова на одних данных → идентичные результаты
- `extractFilePaths` — regex extraction путей файлов
- Пустой/malformed input → graceful result (не throw)

Создать `tests/memory-extract-l1.test.mjs`:
- `buildExtractionPrompt` — формат промпта содержит сессионные данные
- `parseLLMResponse` — валидный JSON → массив кандидатов
- `parseLLMResponse` — JSON внутри markdown → корректный массив
- `parseLLMResponse` — пустой ответ → `[]`
- `parseLLMResponse` — кандидат с пустым content → отфильтрован

Создать `tests/memory-catalog.test.mjs`:
- `generateCatalog` — пустой индекс → пустой catalog
- `selectHotMemories` — фильтрация по проекту, top-30 лимит
- `formatSessionStartOutput` — пустой индекс → пустая строка / placeholder
- `formatSessionStartOutput` — с memories → форматированный вывод без ошибок
</task>
