# Plan 03-02: L0 extraction, hotness, dedup и целостность индекса

## Objective
Расширить L0 extraction новыми полями (agent, duration, commands, errors), исправить hotness tau, сделать dedup агрессивнее с merge стратегией, добавить cleanup tmp файлов при чтении индекса.

## Context
- L0Data сейчас: summary, project, messageCount, files, timestamp. Нужно: + agent, duration, commands, errors, topics
- Детерминированность: timestamp из данных сессии или mtime, не Date.now()
- Hotness tau: 30→60 дней, синхронно в hotness.ts и session-start.ts
- Dedup: FUZZY_THRESHOLD 0.6→0.5, fuzzy match → merge (сейчас skip)
- Atomic write уже реализован, нужен только cleanup stale tmp файлов в readIndex

<task id="03-02-01" title="Расширить интерфейс L0Data новыми полями">
В `src/memory/types.ts` добавить опциональные поля в интерфейс `L0Data`:
- `agent?: string` — идентификатор агента
- `duration?: number` — длительность сессии в мс (разница первого и последнего сообщения)
- `commands?: string[]` — команды/tool invocations из сессии
- `errors?: string[]` — строки ошибок
- `topics?: string[]` — темы сессии

Все поля опциональные — best-effort, не ломать pipeline при отсутствии.
</task>

<task id="03-02-02" title="Обновить L0 extraction парсеры для всех агентов" depends_on="03-02-01">
В `src/memory/extract-l0.ts`:
- `extractL0FromMessages` — добавить extraction команд (искать tool_use, bash blocks), ошибок (error patterns), duration (первый-последний timestamp)
- Передавать agentId в результат

В `src/memory/extract-l0-multi.ts`:
- Каждый агентский парсер (claude, codex, qwen, gemini, companion) проставляет `agent` поле
- Добавить extraction commands/errors специфично для формата каждого агента
- Детерминированность timestamp: брать из данных сессии (первое сообщение) или из mtime файла, НЕ Date.now()

Fixture matrix для валидации — в плане 03-05 (тесты).
</task>

<task id="03-02-03" title="Hotness tau=60 и синхронизация констант">
В `src/memory/hotness.ts`:
- Изменить `DECAY_TAU_DAYS` с 30 на 60

В `src/hooks/session-start.ts`:
- Найти дублирующую константу tau и изменить с 30 на 60
- Убедиться что веса frequency=0.4, recency=0.3, relevance=0.3 совпадают в обоих файлах

Оба изменения в одном коммите для атомарности.
</task>

<task id="03-02-04" title="Dedup: агрессивный threshold + merge вместо skip" depends_on="03-02-01">
В `src/memory/dedup.ts`:
- Снизить `FUZZY_THRESHOLD` с 0.6 до 0.5
- `EXACT_SKIP_THRESHOLD = 0.8` — оставить без изменений

В `src/memory/extract-l1.ts` (или где обрабатывается resolution):
- При `resolution.action === 'fuzzy'` — выполнять merge (`mergeContent`) вместо skip
- Merge: объединить уникальное из обоих воспоминаний, обновить hotness

Убедиться что `resolveCandidate` корректно возвращает action='merge' при fuzzy match.
</task>

<task id="03-02-05" title="Cleanup stale tmp файлов в readIndex">
В `src/memory/index.ts`:
- В функции `readIndex` (или при её вызове) добавить cleanup: если рядом с index файлом есть `*.tmp.*` файлы старше 5 минут — удалить их
- Использовать `statSync` для проверки возраста, `unlinkSync` для удаления
- Ошибки удаления — игнорировать (best-effort cleanup)
- Обернуть в try/catch, не ломать чтение если cleanup упал
</task>
