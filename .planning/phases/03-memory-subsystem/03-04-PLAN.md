# Plan 03-04: extract-memory CLI команда end-to-end

## Objective
Создать CLI команду `extract-memory` с полным pipeline: загрузка сессий → L0 extraction → сохранение в index → запуск L1 extraction. Команда интегрируется в routing через `src/commands/index.ts`.

## Context
- Файл `src/commands/extract-memory.ts` отсутствует — создать с нуля
- Поток: extract-memory → loadSessions() → for each → L0 → store in index.sessions → checkPendingExtractions() → spawn extract-l1
- Аргументы: `--agent <agentId>`, `--session <id>`, `--all`
- Паттерны: Result type, AdapterError, DI через FsDeps
- Exit code 0 на успех, non-zero на ошибку
- Вывод: прогресс + итоговая статистика

<task id="03-04-01" title="Создать src/commands/extract-memory.ts">
Создать файл `src/commands/extract-memory.ts` с основной логикой:

1. **Парсинг аргументов:**
   - `--agent <agentId>` — фильтр по агенту (опционально)
   - `--session <sessionId>` — конкретная сессия (опционально)
   - `--all` — все сессии
   - Без флагов — только pending (сессии без L0/L1)

2. **Pipeline:**
   - Вызвать `loadSessions()` с фильтрами
   - Для каждой сессии: выполнить L0 extraction (`extractL0FromMessages` или мульти-парсер)
   - Сохранить L0 результаты в `index.sessions`
   - Определить pending L1 через `checkPendingExtractions()`
   - Для pending: spawn `extract-l1.ts` (через spawnSync или fork)

3. **Вывод:**
   - Прогресс: `[N/M] Обработка сессии <id>...`
   - Итог: `Обработано: N сессий, извлечено: M воспоминаний`
   - Ошибки: выводить предупреждения, не прерывать pipeline

4. **Error handling:**
   - Обернуть в try/catch, использовать AdapterError для агент-специфичных ошибок
   - Exit code 0 если хотя бы частично успешно, 1 при полном провале

Стиль: функциональный, early returns, комментарии на русском.
</task>
