# Plan 03-01: i18n рефакторинг — разбивка монолита на модули

## Objective
Разбить монолитный `src/core/i18n.ts` (~2132 строки) на логические модули по доменам. Упрощает навигацию, tree-shaking, и дальнейшую поддержку. Прямой импорт без re-export backward compat.

## Context
- Решение из CONTEXT.md: разбить на `common.ts`, `sessions.ts`, `agents.ts`, `cli.ts`, `memory.ts`
- Чистый срез — удалить старый монолит, обновить все импорты за один коммит
- Риск: много импортов `from '../core/i18n.js'` по проекту — нужен полный grep
- Prettier: tabWidth=3, singleQuote=true, printWidth=180

<task id="03-01-01" title="Анализ i18n монолита и составление карты строк по доменам">
Прочитать `src/core/i18n.ts`. Составить карту: какие строки/функции относятся к какому домену (common, sessions, agents, cli, memory). Записать маппинг в комментарий внутри первого нового файла. Также grep все импорты `from.*i18n` по проекту — составить список файлов для обновления.
</task>

<task id="03-01-02" title="Создать модульные файлы i18n" depends_on="03-01-01">
Создать директорию `src/core/i18n/` и файлы:
- `common.ts` — общие строки (today, yesterday, daysAgo, noDescription, msgs, форматирование дат)
- `sessions.ts` — строки сессий (noSessionsFound, recentSessions, searchResults, picker)
- `agents.ts` — строки агентов (agentNotInstalled, resumeError, адаптеры)
- `cli.ts` — CLI usage, help strings
- `memory.ts` — строки подсистемы памяти (extraction, hotness, catalog)
- `index.ts` — barrel re-export из всех модулей (для удобства перехода)

Каждый файл экспортирует именованные функции/объекты. Типы сохранить. Язык определяется так же как в оригинале.
</task>

<task id="03-01-03" title="Миграция импортов по всему проекту" depends_on="03-01-02">
Обновить все импорты во всех файлах проекта с `from '../core/i18n.js'` (и вариации путей) на новые пути из `src/core/i18n/index.js` или конкретных модулей. Один коммит — атомарная миграция. Проверить что `npm run build` (tsc) проходит без ошибок.
</task>

<task id="03-01-04" title="Удалить старый монолит и проверить тесты" depends_on="03-01-03">

</task>
